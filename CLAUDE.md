# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal site for Mike Renoe ([mikerenoe.com](https://www.mikerenoe.com)). A hybrid: the home and run-map pages are hand-written static HTML, and the `/blog` section is generated by **Hugo**. Both are produced into a single `public/` output by one Vercel build. Deployed via `vercel.json`: `buildCommand` is `npm run build && hugo --gc --minify`, `outputDirectory` is `public`, `cleanUrls` serves `/whereirun`, and a rewrite maps `/feed.xml` → the Hugo section feed at `/blog/index.xml`. Custom domains are managed in the Vercel dashboard. Pin Hugo on Vercel with a `HUGO_VERSION` env var (extended).

## Architecture

Hugo runs at the repo root but is **blog-only** — `hugo.toml` sets `disableKinds = ["home", "taxonomy", "term"]` so Hugo never generates a root `index.html`. The hand-written pages live in `static/` and are copied verbatim to the output root, keeping their URLs:

- `static/index.html` — home page.
- `static/whereirun.html` — "Where I Run" interactive US map (served at `/whereirun`).
- `static/images/`, `static/scripts/`, `static/favicon.png`, `static/scss/` — assets.

Hugo owns `content/blog/*.md` → `/blog` (list) + `/blog/<slug>/` (posts) + `/blog/index.xml` (RSS). Templates are in `layouts/_default/`: `baseof.html` (shared shell), `list.html`, `single.html`, `rss.xml` (full-content feed with `<content:encoded>`).

> The header/nav + footer markup is **duplicated** between the static pages and `layouts/_default/baseof.html` (the cost of blog-only scope). When changing nav or shared chrome, update `static/index.html`, `static/whereirun.html`, **and** `baseof.html`.

## Styling workflow

One SCSS source, `scss/index.scss`, is compiled to `static/scss/index.css` (compressed, generated — gitignored, not hand-edited) and shared by both the static pages and the Hugo blog. Edit the `.scss`, then:

- `npm run build` — compile once (also runs first in the Vercel build).
- `npm run watch` — recompile on save.
- `npm run preview` / `npm run dev` — `npm run build && hugo server -D` (local preview incl. drafts; **requires Hugo installed locally** — `brew install hugo`, extended).

The custom `simpel-medium` font is `@font-face`'d from `static/scss/simpel-medium.woff2` (must sit next to the compiled CSS).

## Authoring posts

`hugo new content blog/my-post.md` (uses `archetypes/default.md`), set `draft = false`, write Markdown. Each post is its own canonical source: `single.html` emits `<link rel="canonical">` defaulting to the post's permalink — set the `canonicalURL` front-matter field only when the original lives elsewhere. The full-content RSS at `/feed.xml` is what Medium/dev.to import from.

## Run map (static/whereirun.html)

The map is rendered client-side with Raphael.js. Two vendored scripts must load in order before the inline `<script>`:
1. `static/scripts/raphael.min.js` (vendor library)
2. `static/scripts/us-map-svg.js` — defines the global `usMap` object mapping each state abbreviation to its SVG path string.

All race data lives **inline at the bottom of `static/whereirun.html`** as arrays of state abbreviations by distance: `runMarathon`, `runHalf`, `run10k`, `run5k`, and `runCaz`. To record a new race, add the state's two-letter abbreviation to the appropriate array. `drawMap()` colors each state by the **highest** distance run there (marathon → half → 10k → 5k → caz → not-yet), so order of the if/else chain encodes that priority. Each distance has a matching `*Draw` style object controlling fill color.
